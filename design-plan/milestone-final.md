## Interactivity Usability Justification
> Explain how the interactivity _functionally_ improves the user's experience and helps them accomplish their goals. (i.e. Your interactivity does _more_ than add additional clicks; the interactivity doesn't insert additional barriers for the user when working towards their goals.)
> Explain how your interactivity's design effectively uses affordances, visibility, feedback, and familiarity.
> Write a paragraph (3-4 sentences)
>
> The hamburger button interactivity improves the user's experience by simplifying the navigation experience on mobile devices. It helps the user accomplish their goals by providing clean navigation accross the website. I ensured the hamburger button was visible to users by providing accessible screen reader labels and used an accessible "hidden" class when hiding the button. The hamburger button is in a familiar spot, the top left corner of the header, so users know where to look for it. The button provides feedback by turning a darker green when hovered, signalling that it is interactive. The button creates affordances for the user by the cursor chaning to a pointer when hovering over the button.
> The modals on my trivia page functionally improves the user's experience by providing more information where it wouldn't be appropriate on the main page. So, when user's click on the correct trivia answer, a modal appears that explains why the answer is correct. It helps the users accomplish their goals by allowing them to test their knowledge of 2010's aesthetics. The button provides feedback by turning green if the correct answer is hovered and the cursor switches to pointer, signaling it is interactive. There is similar feedback in the close button: the button turns a lighter color to signal that it is interactive. The modal offers affordances like having a pointer cursor on the button to open it and close it. To ensure visibility, I included a screen reader for each correct answer button that says "Show explanation for why this answer is correct."
